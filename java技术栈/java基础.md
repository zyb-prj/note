### 1.1.2 计算机基础知识

#### 1.1.2.1 二进制

​		计算机中的数据不同于人们生活中的数据，人们生活采用十进制数，而计算机中全部采用二进制数表示，它只包含0、1两个数，逢二进一，1+1=10。每一个0或者每一个1，叫做1个bit（比特）。

- 十进制数据转成二进制数据：使用除以2获取余数的方式

- 二进制数据转成十进制数据：使用8421编码的方式，把所有1位上的十进制求和

#### 1.1.2.2 字节

​		字节是我们常见的计算机中最小存储单元。计算机存储任何的数据，都是以字节的形式存储，右键点击文件属性，我们可以查看文件的字节大小。8个``bit`（二进制位）0000-0000表示为1个字节，写成1`byte`或者1`B`

- 8 `bit`=1 `B`

- 1024 `B`=1 `KB`

- 1024 `KB`=1 `MB`

- 1024 `MB`=1 `GB`

- 1024 `GB`=1 `TB`

### 1.3.3 入门程序说明

#### 1.3.3.1 编译和运行是两回事

- 编译：是指将我们编写的java源文件翻译成JVM认识的class文件，在这个过程中，javac编译器会检查我们所写的程序是否有错误，有错误就会提示出来，如果没有操作就会编译成功。
- 运行：是指将class文件交给JVM运行，此时JVM就会去执行我们编写的程序了。

#### 1.3.3.2 关于main方法

​		main方法称为主方法，写法是固定格式不可以更改。main方法是程序的入口点或起始点，无论我们编写多少程序，JVM在运行的时候，都会从main方法这里开始执行。

### 1.3.4 HelloWorld源代码结构详解

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("hello world!");
    }
}
```

- 第1行的第3个单词必须和所在的文件名完全一样，大小写也要一样
  - `public class`后面代表定义一个类的名称，类是java当中所有源代码的基本组织单位
- 第2行的内容是万年不变的固定写法，代表main方法
  - 这一行代表程序执行的起点
- 第3行代表打印输出语句
  - 希望显示是什么东西，就在小括号中填写什么东西

### 1.3.5 添加注释comment

- 注释：就是对代码的解释说明，其目的是让人们能够更加轻松的了解代码。为代码添加注释，是十分必要的，它不影响程序的编译和运行。
- Java中有单行注释和多行注释
  - 当行注释以`//开头 换行结束`
  - 所行注释以`/*开头 */结束`

## 2.1 数据类型转换

​		java程序中要求参与计算的数据，必须要保证数据类型的一致性，如果数据类型不一致将发生类型的转换。

### 2.1.1 自动转换

#### 2.1.1.1 举例说明

​		一个`int`类型变量和一个`byte`类型变量进行加法运算，结果会是什么类型？

```java
int i = 1;
btye b = 2;
```

​		运算结果，变量的类型将是`int`类型，这就是出现了数据类型的自动类型的转换现象。

#### 2.1.1.2 结论：**将取值范围小的类型自动提升为取值范围大的类型**

- 不同单位的数据进行运算，首先得统一单位
- 例如：大范围`int`、小范围`byte`
  - 先把`byte`类型统一成`int`类型（提升）
  - 然后再和`int`类型进行运算
  - 最后结果是`int`类型

#### 2.1.1.3 转换规则

```java
byte、short、char-->int-->long-->float-->double
```

### 2.1.2 强制转换

#### 2.1.2.1 举例说明

​		 将1.5赋值到`int`类型会编译报错，肯定无法赋值。

​		`double`类型内存8个字节，`int`类型内存4个字节。1.5是`double`类型，取值范围大于`int`。可以理解为`double`是8升的水壶，`int`是4升的水壶，不能把大水壶中的水直接倒入小水壶中去。

​		想要赋值成功，只有通过强制类型转换，将`double`类型强制转换成`int`类型才能赋值。

#### 2.1.2.2 强制类型转换

- 说明：将取值范围大的类型强制转换成取值范围小的类型，**可能会丢失数据**

- 转换格式：

  ```java
  数据类型 变量名 = (数据类型) 被赋值数据;
  ```

#### 2.1.2.3 强烈注意

- 浮点转成整数，直接取消小数点，可能造成数据损失精度。
- `int`强制转换成`short`砍掉两个字节，可能造成数据丢失。

### 2.2.5 三元运算符

  ```java

  ```

- 三元运算符计算方式
  - 布尔类型表达式结果是true，三元运算符整体结果为结果1，赋值给变量
  - 布尔类型表达式结果是false，三元运算符整体结果为结果2，赋值给变量

```java

	int i = (100 == 200 ? 1 : 2);					// i = 2
	int j = (100 == 100 ? 5 : 6);					// j = 5
}
```

## 2.3 方法入门

### 2.3.1 概述

​		我们在学习运算符的时候，都为每个运算符单独的创建一个新的类和main方法，我们会发现这样编写的代码非常繁琐，而且重复的代码过多。规避这种代码就要使用方法来实现。

​		方法就是将一个功能抽取出来，把代码单独定义再一个大括号内，形成一个单独的功能。当我们需要这个功能的时候，就可以去调用。这样既实现了代码的重复性，也解决了代码冗余的现象。

### 2.3.2 调用

​		方法再定义完毕后，方法不会自己运行，必须被调用才能执行，我们可以在主方法main中来调用我们自己定义好的方法。再主方法中，直接写要调用的放大名字就可以调用了。
